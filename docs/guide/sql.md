# SQL Query Builder

The SQL query builder creates parameterized `IDataCommand` instances from fluent builder expressions. It is useful when a query has optional filters, dynamic projections, provider-specific paging, insert/update output clauses, or an upsert shape that is easier to compose than hand-written SQL.

Use it from an `IDataSession` with `session.Sql(builder => ...)`, then execute the returned command with the normal query methods.

```csharp
await using var session = Services.GetRequiredService<IDataSession>();

var users = await session
    .Sql(builder => builder
        .Select<User>()
        .Where(u => u.EmailAddress, "@battlestar.com", FilterOperators.Contains)
        .OrderBy(u => u.EmailAddress)
        .Limit(0, 10)
    )
    .QueryAsync<User>();
```

Builder values are added as command parameters such as `@p0000`. Table names, schemas, and column names are generated by the configured query generator. The SQL shown in this guide uses SQL Server-style quoting and paging; PostgreSQL and SQLite generators produce their own syntax where needed.

## Builder types

`Select<T>()`, `Insert<T>()`, `Update<T>()`, `Delete<T>()`, and `Upsert<T>()` use entity metadata such as `[Table]`, `[Column]`, `[Key]`, `[NotMapped]`, and database-generated properties. They also accept expression-based column selectors.

```csharp
var user = await session
    .Sql(builder => builder
        .Select<User>()
        .Where(u => u.Id, userId)
    )
    .QuerySingleAsync<User>();
```

Use the non-generic builders when you want to supply table and column names directly.

```csharp
var count = await session
    .Sql(builder => builder
        .Select()
        .Count()
        .From("User")
        .Where("IsDeleted", false)
    )
    .QueryValueAsync<int>();
```

## Select rows

When no columns are selected, the builder emits `SELECT *`. Add `Column` or `Columns` calls for projections and aliases.

```csharp
var users = await session
    .Sql(builder => builder
        .Select<User>()
        .Column(u => u.Id)
        .Column(u => u.DisplayName)
        .Column(u => u.EmailAddress)
        .Where(u => u.IsDeleted, false)
        .OrderBy(u => u.DisplayName)
        .Page(page: 1, pageSize: 25)
    )
    .QueryAsync<User>();
```

Example SQL:

```sql
SELECT [Id], [DisplayName], [EmailAddress]
FROM [User]
WHERE ([IsDeleted] = @p0000)
ORDER BY [DisplayName] ASC
OFFSET 0 ROWS FETCH NEXT 25 ROWS ONLY;
```

Use `Limit(offset, size)` for explicit offsets, or `Page(page, pageSize)` for one-based page numbers. If the size or page size is less than or equal to zero, paging is not emitted.

## Filters

`Where` adds a parameterized predicate. The default operator is `FilterOperators.Equal`.

```csharp
var users = await session
    .Sql(builder => builder
        .Select<User>()
        .Where(u => u.EmailAddress, "@battlestar.com", FilterOperators.Contains)
        .Where(u => u.AccessFailedCount, 3, FilterOperators.LessThanOrEqual)
        .Where(u => u.LockoutEnd, null, FilterOperators.IsNull)
    )
    .QueryAsync<User>();
```

Common operators are `Equal`, `NotEqual`, `LessThan`, `LessThanOrEqual`, `GreaterThan`, `GreaterThanOrEqual`, `StartsWith`, `EndsWith`, `Contains`, `IsNull`, `IsNotNull`, and `In`.

Use `WhereIn` for a list of values. Each item is emitted as its own parameter.

```csharp
var statusIds = new[] { 1, 2, 3 };

var statuses = await session
    .Sql(builder => builder
        .Select<Status>()
        .WhereIn(s => s.Id, statusIds)
    )
    .QueryAsync<Status>();
```

Example SQL:

```sql
SELECT *
FROM [dbo].[Status]
WHERE ([Id] IN (@p0000,@p0001,@p0002));
```

Use `WhereIf` and `WhereInIf` for optional query criteria.

```csharp
var users = await session
    .Sql(builder => builder
        .Select<User>()
        .WhereIf(
            u => u.EmailAddress,
            emailFilter,
            FilterOperators.Contains,
            (_, value) => !string.IsNullOrWhiteSpace(value))
        .WhereInIf(
            u => u.Id,
            selectedUserIds,
            (_, values) => values.Any())
    )
    .QueryAsync<User>();
```

Use `WhereOr` and `WhereAnd` to group predicates.

```csharp
var users = await session
    .Sql(builder => builder
        .Select<User>()
        .Where(u => u.EmailAddress, "@battlestar.com", FilterOperators.Contains)
        .WhereOr(or => or
            .Where(u => u.IsDeleted, false)
            .Where(u => u.IsEmailAddressConfirmed, true)
        )
    )
    .QueryAsync<User>();
```

Example SQL:

```sql
SELECT *
FROM [User]
WHERE ([EmailAddress] LIKE '%' + @p0000 + '%' AND ([IsDeleted] = @p0001 OR [IsEmailAddressConfirmed] = @p0002));
```

## Joins

Join builders support table aliases and join types. For typed joins, the entity metadata supplies table and column names.

```csharp
var users = await session
    .Sql(builder => builder
        .Select<User>()
        .Column(u => u.DisplayName, "u")
        .Column(u => u.EmailAddress, "u")
        .Column<Role>(r => r.Name, "r", "RoleName")
        .From(tableAlias: "u")
        .Join<UserRole>(join => join
            .Left(u => u.Id, "u")
            .Right(ur => ur.UserId, "ur")
        )
        .Join<UserRole, Role>(join => join
            .Left(ur => ur.RoleId, "ur")
            .Right(r => r.Id, "r")
        )
        .Where(u => u.EmailAddress, "@battlestar.com", "u", FilterOperators.Contains)
        .OrderBy(u => u.DisplayName, "u")
    )
    .QueryAsync<User>();
```

Example SQL:

```sql
SELECT [u].[DisplayName], [u].[EmailAddress], [r].[Name] AS [RoleName]
FROM [User] AS [u]
INNER JOIN [UserRole] AS [ur] ON [u].[Id] = [ur].[UserId]
INNER JOIN [Role] AS [r] ON [ur].[RoleId] = [r].[Id]
WHERE ([u].[EmailAddress] LIKE '%' + @p0000 + '%')
ORDER BY [u].[DisplayName] ASC;
```

Set the join type with `.Type(JoinTypes.Left)`, `.Type(JoinTypes.Right)`, or another supported `JoinTypes` value.

## Aggregates and grouping

Use `Count` for row counts and `Aggregate` for aggregate functions such as `Sum`, `Average`, `Min`, and `Max`.

```csharp
var activeCount = await session
    .Sql(builder => builder
        .Select<Status>()
        .Count()
        .Where(s => s.IsActive, true)
    )
    .QueryValueAsync<int>();
```

```csharp
var displayOrderTotal = await session
    .Sql(builder => builder
        .Select<Status>()
        .Aggregate(s => s.DisplayOrder, AggregateFunctions.Sum, columnAlias: "TotalDisplayOrder")
        .GroupBy(s => s.IsActive)
    )
    .QueryValueAsync<int>();
```

Example SQL:

```sql
SELECT SUM([DisplayOrder]) AS [TotalDisplayOrder]
FROM [dbo].[Status]
GROUP BY [IsActive];
```

## Insert rows

Use `Value` to add explicit columns, or `Values(entity)` to include mapped, non-generated entity properties.

```csharp
var id = Guid.NewGuid();

var userId = await session
    .Sql(builder => builder
        .Insert<User>()
        .Value(u => u.Id, id)
        .Value(u => u.EmailAddress, $"{id}@email.com")
        .Value(u => u.DisplayName, "Last, First")
        .Output(u => u.Id)
    )
    .QueryValueAsync<Guid>();
```

Example SQL:

```sql
INSERT INTO [User] ([Id], [EmailAddress], [DisplayName])
OUTPUT [INSERTED].[Id]
VALUES (@p0000, @p0001, @p0002);
```

Insert an entity when the values already live in a model.

```csharp
var user = new User
{
    Id = Guid.NewGuid(),
    EmailAddress = "lee.adama@battlestar.com",
    DisplayName = "Adama, Lee",
    Created = DateTimeOffset.UtcNow,
    Updated = DateTimeOffset.UtcNow
};

await session
    .Sql(builder => builder
        .Insert<User>()
        .Values(user)
    )
    .ExecuteAsync();
```

Pass column names to `Values(entity, columnNames)` when only part of an entity should be written.

## JSON values

`ValueJson` serializes a value to JSON before binding it as a command parameter. It is available on insert, update, and upsert builders.

```csharp
var audit = new
{
    Source = "Import",
    Count = 42,
    Created = DateTimeOffset.UtcNow
};

await session
    .Sql(builder => builder
        .Insert()
        .Into("JsonLog")
        .Value("Id", Guid.NewGuid())
        .ValueJson("Data", audit)
    )
    .ExecuteAsync();
```

Example SQL:

```sql
INSERT INTO [JsonLog] ([Id], [Data])
VALUES (@p0000, @p0001);
```

Use the overloads that accept `JsonSerializerOptions` or `JsonTypeInfo<T>` when you need custom serialization or source-generated JSON metadata.

```csharp
await session
    .Sql(builder => builder
        .Update()
        .Table("JsonLog")
        .ValueJson("Data", audit, AuditJsonContext.Default.AuditRecord)
        .Where("Id", logId)
    )
    .ExecuteAsync();
```

## Update rows

Update builders combine `Value`, optional `Output`, and `Where` clauses. Always include a filter unless you intentionally want to update every row.

```csharp
var updatedId = await session
    .Sql(builder => builder
        .Update<User>()
        .Value(u => u.DisplayName, "Updated")
        .ValueIf(u => u.LastLogin, lastLogin, (_, value) => value.HasValue)
        .Output(u => u.Id)
        .Where(u => u.Id, userId)
    )
    .QueryValueAsync<Guid>();
```

Example SQL:

```sql
UPDATE [User]
SET [DisplayName] = @p0000, [LastLogin] = @p0001
OUTPUT [INSERTED].[Id]
WHERE ([Id] = @p0002);
```

For update statements that depend on another table, use `From` and `Join` with aliases.

```csharp
await session
    .Sql(builder => builder
        .Update<User>()
        .Table("User", tableAlias: "u")
        .Value(u => u.IsEmailAddressConfirmed, true)
        .From(tableAlias: "u")
        .Join<UserRole>(join => join
            .Left(u => u.Id, "u")
            .Right(ur => ur.UserId, "ur")
        )
        .Where<UserRole, Guid>(ur => ur.RoleId, roleId, "ur")
    )
    .ExecuteAsync();
```

## Delete rows

Delete builders support typed filters, joins, and output columns. Always include a `Where` clause unless you intentionally want to delete every row.

```csharp
var deletedId = await session
    .Sql(builder => builder
        .Delete<User>()
        .Output(u => u.Id)
        .Where(u => u.Id, userId)
    )
    .QueryValueAsync<Guid>();
```

Example SQL:

```sql
DELETE FROM [User]
OUTPUT [DELETED].[Id]
WHERE ([Id] = @p0000);
```

Use the non-generic builder when the table name is not represented by a mapped entity.

```csharp
await session
    .Sql(builder => builder
        .Delete()
        .Table("JsonLog")
        .Where("Created", cutoff, FilterOperators.LessThan)
    )
    .ExecuteAsync();
```

## Upsert rows

Upsert builders insert a row when the key does not exist and update non-key values when it does. SQL Server output uses `MERGE`; other providers generate equivalent provider-specific SQL.

With explicit table and column names, define at least one key and at least one non-key value.

```csharp
await session
    .Sql(builder => builder
        .Upsert()
        .Into("Status", "dbo")
        .Key("Id")
        .Value("Id", 1)
        .Value("Name", "Upsert Status")
        .Value("Description", "Inserted or updated")
    )
    .ExecuteAsync();
```

Example SQL:

```sql
MERGE INTO [dbo].[Status] AS TARGET
USING (VALUES (@p0000, @p0001, @p0002)) AS SOURCE
([Id], [Name], [Description])
ON TARGET.[Id] = SOURCE.[Id]
WHEN MATCHED THEN
UPDATE SET TARGET.[Name] = SOURCE.[Name], TARGET.[Description] = SOURCE.[Description]
WHEN NOT MATCHED THEN
INSERT ([Id], [Name], [Description])
VALUES (SOURCE.[Id], SOURCE.[Name], SOURCE.[Description]);
```

Typed upserts infer keys from `[Key]` properties when you call `Values(entity)`.

```csharp
[Table("Status", Schema = "dbo")]
public class StatusUpsert
{
    [Key]
    public int Id { get; set; }

    public string Name { get; set; } = null!;

    public string? Description { get; set; }
}

var status = new StatusUpsert
{
    Id = 1,
    Name = "Upsert Entity",
    Description = "Inserted or updated"
};

var statusId = await session
    .Sql(builder => builder
        .Upsert<StatusUpsert>()
        .Values(status)
        .Output(s => s.Id)
    )
    .QueryValueAsync<int>();
```

Use `Key`, `KeyIf`, `Value`, `ValueIf`, and `ValueJson` when you need more control than `Values(entity)`.

```csharp
await session
    .Sql(builder => builder
        .Upsert()
        .Into("JsonLog")
        .Key("Id")
        .Value("Id", logId)
        .ValueJson("Data", audit)
        .Output("Id")
    )
    .QueryValueAsync<Guid>();
```

## Output clauses

`Output` is available on insert, update, delete, and upsert builders. It lets you return generated or affected values and read them with `QueryValue`, `QuerySingle`, or `Query`.

```csharp
var id = await session
    .Sql(builder => builder
        .Insert<User>()
        .Values(user)
        .Output(u => u.Id)
    )
    .QueryValueAsync<Guid>();
```

Use `OutputIf` when output should be included only for specific flows.

```csharp
var command = session.Sql(builder => builder
    .Update<User>()
    .Value(u => u.DisplayName, displayName)
    .OutputIf(u => u.Id, condition: _ => returnUpdatedId)
    .Where(u => u.Id, userId));
```

## Raw statements and mixed batches

Use `Statement()` to include raw SQL in the same builder batch. Parameters added to a statement are shared with the generated statements in the batch.

```csharp
var values = new[] { 1, 2, 3 }.ToDelimitedString();

var statuses = await session
    .Sql(builder =>
    {
        builder
            .Statement()
            .Query("CREATE TABLE #identifiers (Id int);");

        builder
            .Statement()
            .Query("INSERT INTO #identifiers (Id) SELECT CONVERT(int, value) FROM STRING_SPLIT(@Identifiers, @Separator);")
            .Parameter("@Identifiers", values)
            .Parameter("@Separator", ",");

        builder
            .Select<Status>()
            .From(tableAlias: "s")
            .Join(join => join
                .Left("Id", "s")
                .Right("Id", "#identifiers", null, "i"));
    })
    .QueryAsync<Status>();
```

Generated output:

```sql
CREATE TABLE #identifiers (Id int);
INSERT INTO #identifiers (Id) SELECT CONVERT(int, value) FROM STRING_SPLIT(@Identifiers, @Separator);
SELECT *
FROM [dbo].[Status] AS [s]
INNER JOIN [#identifiers] AS [i] ON [s].[Id] = [i].[Id];
```

Raw helpers are also available for selected clauses, including `FromRaw`, `WhereRaw`, and `OrderByRaw`. Prefer the typed and parameterized methods for user input, and reserve raw clauses for trusted SQL fragments.

## Tags and comments

`Tag()` adds a provider-formatted SQL comment with the caller name, source file, and line number. This is useful when query logging is enabled.

```csharp
var statuses = await session
    .Sql(builder => builder
        .Select<Status>()
        .Where(s => s.IsActive, true)
        .Tag()
    )
    .QueryAsync<Status>();
```

Example SQL:

```sql
/* Caller; GetStatusesAsync() in StatusRepository.cs:line 42 */
SELECT *
FROM [dbo].[Status]
WHERE ([IsActive] = @p0000);
```

Use `Tag("Custom label")` for a custom caller label, or `Comment("text")` when you want only a literal SQL comment.

## Execution

The builder only creates the command text and parameters. The returned `IDataCommand` can be executed with the same APIs as hand-written SQL.

```csharp
await session
    .Sql(builder => builder
        .Update<User>()
        .Value(u => u.LastLogin, DateTimeOffset.UtcNow)
        .Where(u => u.Id, userId)
    )
    .ExecuteAsync();
```

```csharp
var users = await session
    .Sql(builder => builder
        .Select<User>()
        .Where(u => u.IsDeleted, false)
    )
    .QueryAsync<User>();
```

```csharp
var json = await session
    .Sql(builder => builder
        .Select<User>()
        .Where(u => u.IsDeleted, false)
    )
    .QueryJsonAsync();
```

You can still add normal command options after the builder call, such as `CommandTimeout`, additional parameters for raw statements, or other `IDataCommand` configuration.
